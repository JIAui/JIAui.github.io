安装Hexo

环境：Node.js、Git.

安装Hexo：使用npm(Node Package Manager)安装Hexo

npm install -g hexo -cli或者使用 npm install hexo 进行局部安装.

安装以后可以使用以下两种方法执行hexo：

  • npm hexo <command>

  • 将Hexo所在目录下的node_modules添加到环境变量中可以直接使用npm hexo <command>:

安装 Hexo 完成后，请执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件.

新建完成后，指定文件夹的目录如下：

_config.yml

Hexo文件夹下中的_config.yml为网站的 配置 信息.

Hexo/themes/landscape中的_config.yml为主题的配置信息.

scaffolds

模版 文件夹。当新建文章时，Hexo 会根据 scaffold 来建立文件。

Hexo的模板是指在新建的文章文件中默认填充的内容。例如，如果修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。

source

资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes

主题 文件夹.Hexo会根据主题来生成静态页面.

创建新文章

根目录命令行输入hexo new <mode> <Title>新建文章.

其中mode在scaffolds中，默认又三个模板post、draft、page

参数 功能 路径
post(default) 新建文章 /source/_posts/
draft 新建草稿 /source/_drafts/
page 新建页面(标签页、分类页等) /source/

通常情况下使用hexo new post即可，注意这里的title不是文章名，具体文章中显示的文章标题可以在文章文件中修改.

例如：命令行输入hexo new article.

此时在source/_posts/文件夹中多了一个article.md文件.
